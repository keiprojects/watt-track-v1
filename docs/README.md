# WattTrack Docs

This folder is the source of truth for implementation planning, feature scope, and delivery status.

## Primary documents

- `spec-plan.md` is the canonical implementation spec derived from the PRD.
- `development-plan.md` is the canonical execution plan for roadmap, milestones, sequencing, and status.

## Working agreement

- If product scope changes, update the PRD and `spec-plan.md` together.
- If delivery order or implementation status changes, update `development-plan.md`.
- If a product or engineering decision changes behavior, update `spec-plan.md` and `development-plan.md` together when relevant.
- If code and planning docs disagree, treat that as a gap to resolve immediately instead of letting both drift.

## Current document set

- `spec-plan.md`: implementation-ready product, feature, data, validation, and architecture spec.
- `development-plan.md`: PRD-aligned execution plan, architecture direction, milestone checklist, and acceptance tracking.
- `privacy-policy.md`: draft user-facing privacy policy for store publication.
- `store-compliance.md`: Google Play and Apple App Store compliance worksheet for release preparation.
- `native-qa-checklist.md`: production-equivalent Android build QA checklist.
- `../site/privacy/index.html`: public privacy policy page source for Coolify static hosting.
- `../site/support/index.html`: public support page source for Coolify static hosting.
